🛠️ Como Contribuir e Pesquisar
1. Pesquisa e Consulta
Utilize a pasta 10_Projetos para entender como as soluções foram arquitetadas. Sinta-se à vontade para propor melhorias nos arquivos existentes via Pull Request.

2. Espaço de Descobertas (99_RESEARCH)
Para novas descobertas, ferramentas ou rascunhos de testes, utilize a pasta 99_RESEARCH.

Crie uma subpasta com seu nome (ex: 99_RESEARCH/nome_colaborador/).

Documente seus achados em Markdown seguindo nossos Templates.

3. Workflow de Promoção (Audit-Ready)
Este repositório segue o princípio de Governança-as-Code. Nenhuma alteração é final até que passe pela auditoria técnica:

Input: Você realiza a pesquisa/teste no ambiente DEV.

Review: O Auditor (Paulo) revisa a qualidade técnica e conformidade.

Promoção: Conteúdos validados são "promovidos" para o Fiqueok_Brain (PROD), que é o nosso repositório oficial e privado.

⚠️ Regras de Ouro
Não exponha segredos: Nunca suba senhas ou tokens em texto plano. Utilize variáveis de ambiente ou o nosso cofre (Vault).

Documente o "Porquê": Não registre apenas o comando; explique o risco que ele mitiga ou o benefício de negócio gerado [cite: 2025-12-02].

Referência de Governança: Para instruções sobre acessos, infraestrutura mínima e tokens, consulte o repositório de Governança do Living Lab.
